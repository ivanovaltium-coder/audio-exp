# Акустическая система обнаружения БПЛА

## Описание проекта

Научная разработка системы акустического обнаружения беспилотных летательных аппаратов (БПЛА/дронов) с использованием конденсаторных микрофонов и звуковой карты профессионального уровня.

### Оборудование

- **Звуковая карта**: Steinberg UR44C (ASIO драйвер)
- **Микрофоны**: 4 × Behringer ECM8000 (всенаправленные измерительные конденсаторные)
- **Конфигурация**: 
  - 8 входных каналов (4 микрофонных с фантомным питанием +48V, 2 линейных, 2 дублирующих)
  - Частота дискретизации: 48 кГц (поддерживается до 96 кГц)
  - Разрядность: 24 бит

### Архитектура системы

```
┌─────────────────────────────────────────────────────────────┐
│                     АКУСТИЧЕСКАЯ СИСТЕМА                    │
├─────────────────────────────────────────────────────────────┤
│  Микрофоны (4× ECM8000) → Steinberg UR44C → ASIO Драйвер   │
│                          ↓                                   │
│  Запись аудио (8 каналов) → Предобработка → Извлечение признаков │
│                          ↓                                   │
│  MFCC признаки → Классификатор (ML) → Решение: ДРОН/ФОН    │
└─────────────────────────────────────────────────────────────┘
```

## Установка

### Требования

- Python 3.10+
- Windows 10/11
- Установленные драйверы Steinberg UR44C (ASIO)
- Виртуальное окружение (рекомендуется)

### Шаги установки

```bash
# Создание виртуального окружения
python -m venv .venv
.venv\Scripts\activate  # Windows

# Установка зависимостей
pip install -r requirements.txt

# Проверка работы ASIO
python main.py --list-asio
```

## Быстрый старт

### 1. Проверка оборудования

```bash
# Показать все ASIO устройства (рекомендуется для UR44C)
python main.py --list-asio

# Проверить работу микрофона
python main.py --check-mic

# Проверка с конкретным устройством
python main.py --check-mic --device 53
```

### 2. Распознавание в реальном времени

```bash
# Стандартный режим
python main.py

# Режим с минимальной задержкой (callback)
python main.py --callback

# С выбором конкретного устройства
python main.py --device 53 --callback
```

### 3. Распознавание из файла

```bash
python main.py --file path/to/audio.wav
```

## Конфигурация

Все настройки находятся в файле `config.py`:

```python
# Аудио параметры
SAMPLE_RATE = 48000           # Частота дискретизации (Гц)
WINDOW_SEC = 3.0              # Длительность окна анализа (сек)
NUM_CHANNELS = 8              # Количество каналов записи
USE_SINGLE_CHANNEL = True     # Использовать 1 канал (версия 1)
ACTIVE_CHANNEL = 0            # Индекс активного канала (0-7)

# ASIO настройки
USE_ASIO = True               # Использовать ASIO драйвер
ASIO_DEVICE_NAME = "Steinberg UR44C ASIO"
ASIO_BUFFER_SIZE = 1024       # Размер буфера (меньше = меньше задержка)

# Параметры MFCC
N_MFCC = 13
N_FFT = 2048
HOP_LENGTH = 512
```

## Режимы работы

### Первая версия (1 микрофон)

По умолчанию используется один микрофон:
- `USE_SINGLE_CHANNEL = True`
- `ACTIVE_CHANNEL = 0` (первый канал)

### Будущая версия (4 микрофона для пеленгации)

Для активации многоканального режима:
```python
USE_SINGLE_CHANNEL = False  # Использовать все каналы
NUM_CHANNELS = 4            # или 8 для полной конфигурации
```

## Команды интерфейса

| Команда | Описание |
|---------|----------|
| `--list-devices` | Показать все аудиоустройства с Host API |
| `--list-asio` | Показать только ASIO устройства (рекомендуется) |
| `--check-mic` | Проверить работу микрофона |
| `--mic-duration 5.0` | Длительность проверки микрофона (сек) |
| `--device 53` | Выбрать устройство по ID |
| `--callback` | Режим минимальной задержки |
| `--file audio.wav` | Распознать файл вместо микрофона |

## Диагностика

### Проблема: ASIO устройство не найдено

**Решение:**
1. Убедитесь, что драйверы Steinberg установлены
2. Закройте другие программы, использующие аудио (DAW, VoiceMeeter)
3. Перезапустите службу аудио Windows
4. Проверьте подключение USB

```bash
python main.py --list-asio
```

### Проблема: Тихий сигнал микрофона

**Решение:**
1. Включите фантомное питание +48V на UR44C
2. Увеличьте gain на входном канале
3. Проверьте подключение XLR кабеля
4. Говорите ближе к микрофону при тесте

```bash
python main.py --check-mic --mic-duration 5.0
```

### Проблема: Ошибка количества каналов

Система автоматически определяет доступное количество каналов и использует максимально возможное.

## Научная основа

Система использует:
- **MFCC (Mel-frequency cepstral coefficients)** — 13 коэффициентов для представления спектра
- **Машинное обучение** — классификатор (Random Forest / SVM / Neural Network)
- **Фазовый анализ** — для пеленгации источника звука (будущая версия)
