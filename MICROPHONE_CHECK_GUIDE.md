# Инструкция по проверке микрофона и настройке записи

## Быстрый старт

### 1. Проверка работы микрофона

Запустите проверку микрофона для подтверждения, что звук записывается именно с конденсаторного микрофона через звуковую карту Steinberg UR44C:

```bash
python main.py --check-mic
```

Или с указанием длительности теста (в секундах):

```bash
python main.py --check-mic --mic-duration 5
```

**Что происходит во время проверки:**
- Записывается короткий фрагмент звука (2 секунды по умолчанию)
- Анализируется уровень сигнала в децибелах (dBFS)
- Если сигнал выше порога -40 дБ, микрофон считается работающим
- Выводятся рекомендации при отсутствии сигнала

### 2. Просмотр доступных аудиоустройств

Для выбора конкретного устройства ввода (звуковой карты):

```bash
python main.py --list-devices
```

Найдите в списке вашу звуковую карту Steinberg UR44C и запомните её номер.

### 3. Запуск с выбранным устройством

```bash
python main.py --check-mic --device 2
```

где `2` — номер устройства из списка.

## Настройка количества каналов

### Для первой версии (1 микрофон)

В файле `config.py` установлены параметры:

```python
NUM_CHANNELS = 4  # запись на 4 канала (возможность на будущее)
USE_SINGLE_CHANNEL = True  # используется только 1 канал
ACTIVE_CHANNEL = 0  # индекс активного канала (0-3)
```

**Как это работает:**
- Звуковая карта записывает на 4 канала одновременно
- Но для анализа используется только канал №0 (первый микрофон)
- Остальные каналы игнорируются в текущей версии

### Для будущей версии (4 микрофона)

Когда будете готовы использовать все 4 микрофона:

1. Измените в `config.py`:
   ```python
   USE_SINGLE_CHANNEL = False  # использовать все каналы
   ```

2. Модифицируйте код обработки для работы с многоканальным сигналом

## Выбор активного канала

Если ваш микрофон подключён к другому входу звуковой карты:

```python
ACTIVE_CHANNEL = 1  # или 2, или 3 (номер входа 0-3)
```

## Порог чувствительности микрофона

В `config.py` можно настроить порог обнаружения сигнала:

```python
MIC_CHECK_THRESHOLD_DB = -40.0  # дБ
```

- Уменьшите значение (например, -50), если микрофон слишком тихий
- Увеличьте значение (например, -30), если много ложных срабатываний

## Рекомендации по подключению

1. **Фантомное питание**: Убедитесь, что на микрофон Behringer ECM8000 подаётся +48V
2. **Подключение**: Микрофон должен быть подключён к входному каналу звуковой карты Steinberg UR44C
3. **Усиление**: Отрегулируйте gain на звуковой карте для оптимального уровня сигнала
4. **Проверка**: Говорите в микрофон во время теста --check-mic

## Примеры использования

```bash
# Проверка микрофона на устройстве по умолчанию
python main.py --check-mic

# Проверка с выбором устройства
python main.py --check-mic --device 2

# Проверка с длительным тестом (5 секунд)
python main.py --check-mic --mic-duration 5

# Запуск распознавания с микрофона
python main.py --device 2

# Распознавание из файла
python main.py --file recording.wav
```
